1. Introduction

EARS (Engagement Activity Recording System) is a modern web-based platform designed for recording, managing, and reporting academic and professional engagement activities.

The system supports:

Online usage via browser
Offline usage via installation (Progressive Web App)
Automatic and manual data synchronization
Admin review and reporting workflows
2. Getting Started
System Requirements
Google Chrome (recommended)
Internet required for first-time setup and syncing
Desktop or laptop recommended (mobile supported)
Access

Open:
👉 https://earsystem.netlify.app/

3. 🔌 Installing EARS as a Chrome Web App (Offline Access)

EARS can be installed as a Progressive Web App (PWA) for offline use.

3.1 Why Install the App?

Installing allows you to:

Use EARS without internet
Run it like a desktop application
Access faster startup
Store data locally
Sync when online
3.2 Method 1 — Install Button (Recommended)
Open Chrome
Go to: https://earsystem.netlify.app/
Look at the right side of the address bar
Click the Install icon (⬇ or computer icon)
Click Install

✔ The app will now appear as a desktop application.

3.3 Method 2 — Chrome Menu Installation

If the install button is not visible:

Open the website in Chrome
Click ⋮ (top-right menu)
Select:
“Install EARS” OR
“Add to Desktop”
Confirm installation
3.4 Method 3 — Automatic Prompt
Chrome may show an automatic popup:
“Install EARS?”
Click Install
3.5 After Installation
App appears on desktop / start menu
Opens in standalone window
No browser tab required

12. 📴 Offline Mode Usage (NEW — IMPORTANT)
12.1 How Offline Mode Works

Once installed as a PWA:

The system stores essential files locally
Core features remain accessible without internet
Data is temporarily saved on the device
12.2 How to Use Offline Mode
Open installed EARS app
Turn off internet (or disconnect Wi-Fi)
Continue using the system normally
Add or edit activities as needed
12.3 Offline Capabilities

Available offline:

Viewing dashboard
Recording activities
Editing saved data
Viewing timeline (cached data)
12.4 Offline Limitations

Not available offline:

Cloud syncing
Admin approvals
Real-time updates
Google Sheets export
13. 🔄 Data Sync System (NEW — CORE FEATURE)
13.1 What is Sync?

Sync ensures:

Offline data is uploaded to server
Cloud data is updated locally
Multi-device consistency
13.2 When Sync Happens
Automatically when internet returns
On app startup (if online)
Manually via sync button
13.3 Sync Workflow
Work offline (if needed)
Reconnect to internet
Open EARS app
System automatically syncs data
13.4 Manual Sync
Go to Admin Panel → Sync Settings
Click: “🔄 Manual Sync Now”
Wait for confirmation
13.5 Sync Safety Rules
Do not close app during sync
Ensure stable internet
Avoid repeated rapid syncing
Check audit logs if issues occur
14. FAQ & Troubleshooting (UPDATED)
Q: I don’t see the install button
Use Chrome menu → “Install EARS”
Refresh page
Ensure you are on Chrome browser
Q: App works offline but data is missing
Data may not have synced yet
Reconnect internet and open app
Wait for sync completion
Q: Sync is not working
Check internet connection
Click “Test Connection” in Admin Panel
Retry manual sync
Q: Can I use EARS permanently offline?
Yes, but only local features work
Cloud backup requires internet sync
